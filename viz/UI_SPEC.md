# Transformer Attention Explorer — UI Specification

Purpose:
Create a beautiful, interactive visualization tool for exploring transformer attention weights generated by a Python model.

The visualization should feel like a **modern ML interpretability tool**, not a basic chart.

Examples of inspiration:
- BertViz
- OpenAI attention visualizers
- Anthropic interpretability tools

---

# Data Source

The visualization reads from:

attn.json

Example format:

{
  "tokens": [12, 3, 7, 19, 2, 5, 44, 11],
  "attention": [[[...]]]
}

Meaning:

tokens → sequence tokens

attention → shape:

[heads][query_token][key_token]

Example:

attention[head][i][j]

= how much token i attends to token j.

---

# Core Features

The UI must include:

1. Attention heatmap
2. Token display
3. Attention head selector
4. Hover tooltips showing values
5. Smooth transitions when switching heads

---

# Layout

Top section:

Title

Transformer Attention Explorer

Short description explaining:

"Each cell shows how strongly one token attends to another."

---

Below title:

Token row visualization

Example:

[The] [cat] [sat] [down]

Tokens should appear as rounded boxes.

---

Main visualization area:

Interactive heatmap.

Axes:

X axis → Key token  
Y axis → Query token

Color scale:

Light → weak attention  
Dark red → strong attention

---

# Interaction

User should be able to:

Select attention head:

Head 0  
Head 1  
Head 2  
Head 3

Changing head updates heatmap smoothly.

Hovering a cell should display:

Query token  
Key token  
Attention weight

Example:

Token 3 attends to Token 1  
Weight: 0.62

---

# Visual Style

Use modern design:

- clean typography
- minimal color palette
- subtle shadows
- rounded UI elements

Colors:

Background: very light gray (#fafafa)

Primary accent: deep red attention scale.

---

# Libraries Allowed

Preferred:

Plotly.js  
D3.js

Avoid heavy frameworks like React unless necessary.

This must run from a **single static HTML file**.

T = 64 tokens

without lag.

# Design Philosophy

The goal is to make the transformer attention **intuitive and explorable**.

A user should be able to quickly see:

- which tokens attend to which
- how patterns differ across heads
- where attention concentrates
